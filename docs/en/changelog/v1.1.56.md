# v1.1.56: Task Rendering Fix, Smoother Windows App Opening

**Release Date**: February 11, 2026

---

## One-Sentence Summary

This is a desktop experience fix release: Task tool rendering issue fixed, Windows now supports opening apps with executables, sidebar no longer auto-collapses when switching sessions.

---

## Bug Fix: Task Tool Rendering Abnormal in Desktop

### Previous Issue

In the desktop app, Task tool related content had rendering abnormalities, affecting reading and operation.

### Current Status

v1.1.56 fixed the rendering issue, Task execution process displays more stably.

### How to Use?

👉 **Full Tutorial**: [Agent Permissions and Security](/en/5-advanced/02c-agent-permissions)

👉 **Official Docs**: [Commands / Subtask](https://opencode.ai/docs/commands#subtask)

**Quick Experience**:

1. Initiate a request requiring subtasks in desktop.
2. Observe if Task output and step rendering are normal.
3. Trigger repeatedly in the same session to confirm display stability.

---

## New Feature: Windows Supports Opening Apps via Executable

### What's Updated?

Desktop added the ability to open apps via executable programs in Windows.

### What's the Use for You?

Local tool integration is more direct, no need to go through many manual steps, especially practical in enterprise environments.

### How to Use?

👉 **Full Tutorial**: [2.1 Interface and Operations](/en/2-daily/01-interface)

**Quick Experience**:

1. Open relevant entry in Windows desktop.
2. Select target app or file and trigger open.
3. Confirm system uses expected executable to start.

---

## Improvement: Switching Sessions No Longer Auto-Collapses Sidebar

### What's Updated?

When switching sessions, sidebar state is preserved and won't be automatically closed.

### What's the Use for You?

Operations are more coherent when switching between multiple sessions back and forth, especially suitable for workflows requiring frequent context comparison.
